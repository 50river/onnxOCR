📖 テスト詳細

フォーム操作テスト

  • 基本的なフォーム入力機能
  • フィールドバリデーション（日付、金額、支払先、適用）
  • フィールド更新とデータバインディング
  • 信頼度インジケーターの表示
  • フォームリセット機能

候補選択機能テスト

  • 候補リストの表示と非表示
  • 候補選択によるフィールド値の更新
  • 候補履歴の管理（追加、重複防止）
  • 候補フィルタリング（信頼度、重複除去）

アクセシビリティテスト

  • ARIA属性の適切な設定
  • キーボードナビゲーション対応
  • スクリーンリーダー対応（ライブリージョン、ラベル）
  • フォーカス管理
  • カラーコントラストと視覚的表示

エラーハンドリングテスト

  • 無効なデータの処理
  • 存在しないDOM要素の処理
  • イベントリスナーのエラー処理

📚 使用方法

テスト実行手順

  1. 要件確認: 上記の要件確認チェックリストで前提条件を確認
  2. テスト実行: 「全テストを実行」ボタンをクリック
  3. 結果確認: テスト出力とサマリーで結果を確認
  4. 個別テスト: 必要に応じて個別のテストカテゴリを実行

自動実行

URLパラメータ ?run-ui-tests を追加すると、ページ読み込み時に自動でテストが実行されます。

例: ui-components-test-runner.html?run-ui-tests

テスト結果の解釈

  • ✅ 合格: テストが正常に完了し、期待される動作を確認
  • ❌ 失敗: テストが失敗し、問題が検出された
  • 成功率: 全テスト中の合格率（90%以上が推奨）
